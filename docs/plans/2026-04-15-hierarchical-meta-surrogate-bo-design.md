# Spec: Hierarchical Meta-Surrogate Bayesian Optimization for Protein Evolution

## 1. Overview
This document outlines the architectural design for upgrading the `evolution` mode in the `protein_pipeline` system. The goal is to replace the naive, zero-knowledge RandomForest Bayesian Optimization (BO) loop with a **Hierarchical Meta-Surrogate BO** approach. This new architecture leverages pre-computed metrics and a pre-trained Deep Neural Network (DNN) backbone to drastically reduce the number of expensive AlphaFold2 (Oracle) evaluations required during sequence optimization.

## 2. Motivation (Paper Defense Strategy)
Traditional BO in sequence design starts with zero biological context, requiring many expensive oracle calls (AF2) to find optimal sequences. While large Protein Language Models (PLMs) like ESM-3 can predict structures without MSAs, they lack the specific, real-time tuning capabilities needed for custom target optimization. 

Our approach introduces a **Cascaded Filtering** system:
1. **Computational Economy:** Filter out non-viable sequences using extremely cheap metrics (e.g., SoluProt solubility) before any structural prediction.
2. **Prior-Informed Surrogate:** Utilize a pre-trained surrogate model (trained on ~1,000 diverse protein families with PLM embeddings) to predict pLDDT scores and filter the pool further.
3. **High-Fidelity Oracle:** Only the most promising candidates are sent to AF2. The exact AF2 results are then used to dynamically fine-tune the surrogate model (Transfer Learning) for the specific target space.

This proves that our system orchestrates state-of-the-art models intelligently, achieving Ground Truth (AF2) accuracy with a fraction of the computational cost.

## 3. Architecture Design: The 3-Stage Pipeline

When a user initiates `evolution_mode`, the sequence generation (e.g., RFDiffusion + ProteinMPNN) creates an initial pool of candidate sequences. These sequences then pass through three gating stages:

### Stage 1: Physicochemical & Solubility Gating (Ultra-Fast)
*   **Input:** Raw amino acid sequences.
*   **Mechanism:** Heuristic checks (length, charge) and fast ML predictions (e.g., SoluProt score).
*   **Action:** Sequences failing predefined thresholds (e.g., SoluProt score < 0.5) are instantly discarded.
*   **Compute Cost:** CPU-only, milliseconds per sequence.

### Stage 2: Meta-Surrogate Pre-Screening (Fast)
*   **Input:** Sequences that passed Stage 1.
*   **Mechanism:** 
    *   Convert sequences into semantic embeddings using a lightweight PLM (e.g., ESM-2).
    *   Pass embeddings through a Deep Surrogate Model pre-trained on 1,000 diverse protein families to map embeddings to expected AF2 pLDDT scores.
*   **Action:** Rank sequences by predicted pLDDT. Only the top $N$ sequences (Acquisition function output) are allowed to proceed.
*   **Compute Cost:** Light GPU inference, <1 second per sequence.

### Stage 3: The Oracle & Feedback Loop (Slow, High-Fidelity)
*   **Input:** Top $N$ sequences from Stage 2.
*   **Mechanism:** Execute standard AlphaFold2 prediction to obtain Ground Truth structure and actual pLDDT.
*   **Action (Feedback):** The actual pLDDT scores are fed back into the BO loop. The Meta-Surrogate model is *fine-tuned* (or acts as the prior for a localized Gaussian Process/DNN) on these specific results, making Stage 2 highly accurate for this specific user target in the next BO round.
*   **Compute Cost:** Heavy GPU inference, minutes per sequence.

## 4. Implementation Plan

### 4.1. Offline Phase: Backbone Model Creation
1.  **Data Collection:** Compile a dataset of ~1,000 diverse protein families.
2.  **Feature Extraction:** Generate PLM embeddings (e.g., ESM-2) for all sequences.
3.  **Label Generation:** Run AF2 on all sequences to get ground truth pLDDT scores.
4.  **Training:** Train a lightweight DNN (e.g., MLP) that maps ESM-2 embeddings to pLDDT scores. Save the weights as `meta_surrogate_v1.pt`.

### 4.2. Online Phase: `pipeline-mcp/src/pipeline_mcp/evolution.py` Update
1.  **Integrate Stage 1:** Before the BO loop starts, filter the initial pool generated by `num_seq_per_tier` using the existing `soluprot` results (which are currently just calculated but not aggressively used as a hard gate for BO).
2.  **Integrate Stage 2:** Replace `RandomForestRegressor` with the loaded `meta_surrogate_v1.pt`. Modify the `calculate_acquisition` function to utilize the DNN's predictions.
3.  **Integrate Stage 3:** Ensure the feedback loop updates the localized surrogate model dynamically during the rounds specified by `evolution_rounds`.

## 5. Success Metrics (For Paper Benchmarking)
*   **Sample Efficiency:** Show the learning curve (Max pLDDT vs. AF2 calls). The new system should reach >85 pLDDT in <5 AF2 calls, whereas standard BO might take 20+.
*   **Compute Time Saved:** Total GPU hours consumed per evolution run compared to naive BO.
*   **Robustness:** Demonstrate performance on a novel target protein not present in the 1,000-family training set.
