# Reproducing Paper Data, Tables, and Figures

The package includes the benchmark inputs and cached results used by the current
manuscript draft:

- `data/benchmark/cath_pilot_dataset.csv`
- `data/benchmark/cath_pilot_emb_320d.npy`
- `data/benchmark/results/`
- `data/case_studies/`
- `data/cath_curated/`
- `data/cath_73/`
- `figures/benchmark/`

## Fast Reproduction From Cached Results

Install dependencies:

```bash
python -m pip install -r requirements-benchmark.txt
```

Regenerate summary tables and figures:

```bash
bash scripts/reproduce_paper_tables_figures.sh
```

This reads the included benchmark result files and rewrites outputs under
`data/benchmark/results/` and `figures/benchmark/`.

The representative 3RGK/1LVM multi-round summaries in `data/case_studies/` are
cached production-run exports. They document the completed execution traces used
in Section 3.8 of the manuscript and are not regenerated by the benchmark figure
script.

The backbone/ensemble ablation is a live pipeline experiment rather than a
cached model benchmark. Use `scripts/benchmark/13_run_backbone_ensemble_ablation.py`
to launch the paired single-backbone, RFD3-single, and RFD3-ensemble arms, then
use `scripts/benchmark/14_make_backbone_ensemble_ablation.py` to summarize
completed outputs. The default pilot uses three CATH targets and keeps the AF2
budget matched across arms. For a long server run, use
`scripts/benchmark/launch_backbone_ensemble_ablation.py`; it records state and
logs under `_ops/jobs/`.

`data/cath_curated/` contains the QC-filtered publication-grade CATH benchmark
subset used by the manuscript. `data/cath_73/` contains raw lightweight
summaries before paper-level QC. The full archive is staged outside the source package under
`/opt/protein_pipeline/cath_outputs` and should be deposited as a large artifact
or S3-backed dataset for publication.

## Full Benchmark Rerun

A full rerun starts from a local or downloaded CATH pipeline-output archive:

```bash
python scripts/benchmark/00_prepare_data.py
python scripts/benchmark/01_compute_embeddings.py --model 8M
python scripts/benchmark/02_model_comparison.py
python scripts/benchmark/03_sample_size_ablation.py
bash scripts/reproduce_paper_tables_figures.sh
```

`01_compute_embeddings.py` downloads ESM-2 models through `transformers` if they
are not already cached. The current manuscript benchmark uses ESM-2 8M
embeddings. The 150M embedding-size ablation script remains available as an
optional supplementary experiment, but a matching 640-D cache is not included in
this release.

To rebuild the benchmark from the full CATH archive, download or mount the full
run-output archive, then point `cath_outputs/` to the QC-included runs listed in
`data/cath_curated/included_runs.txt`. Run `00_prepare_data.py`, regenerate
embeddings, and rerun all downstream result tables. Do not train on the excluded
fallback records: they are input/design-generation failures for this pipeline
contract, not valid pLDDT = 0 design examples.

## Benchmark Script Roles

- `00_prepare_data.py` builds the master CATH pilot CSV from `cath_outputs/`.
- `01_compute_embeddings.py` computes ESM-2 embeddings.
- `02_model_comparison.py` compares surrogate model families.
- `03_sample_size_ablation.py` evaluates active-learning training sample counts.
- `04_esm_size_ablation.py` optionally compares ESM embedding sizes when both
  320-D and 640-D caches match the active benchmark CSV.
- `05_aggregate_and_test.py` creates summary and statistical test tables.
- `06_make_figures.py` regenerates core benchmark figures.
- `07_mpnn_error_decomposition.py` estimates MPNN/SoluProt/AF2 error components.
- `09_bias_analysis.py` evaluates selection bias.
- `10_ensemble_benchmark.py` evaluates rank-mean surrogate ensembles.
- `11_make_method_figures.py` regenerates method figures, including the active-learning loop.
- `12_make_cath_curated_figure.py` regenerates the QC-filtered CATH benchmark figure and summary table.
- `13_run_backbone_ensemble_ablation.py` runs the matched backbone/ensemble ablation pilot.
- `14_make_backbone_ensemble_ablation.py` summarizes completed backbone/ensemble ablation outputs.
- `launch_backbone_ensemble_ablation.py` starts the ablation pilot as a managed background job.

The scripts in this public package calculate `PROJECT_ROOT` from their own file
locations, so they no longer depend on the original development checkout path.
